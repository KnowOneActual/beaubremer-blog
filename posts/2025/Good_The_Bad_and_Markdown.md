Have you ever written on GitHub or Reddit? If so, you have used Markdown. This simple text format is the standard for
web writing.

All tools have pros and cons. This post covers Markdown benefits, common mistakes, and quick tips.

### The pros of Markdown

Markdown is popular for good reasons. It helps writers and bloggers.

- **Simple and readable**: Markdown is easy to read as plain text. Anyone can understand bold or italic styles. Simple
  formatting helps you focus on your writing.
- **Works anywhere**: Markdown files are plain text. You can edit them on any device or app. Your content is never
  locked in Word files.
- **Easy to track**: Markdown works well with Git. You can track changes and work with others easily.
- **Fast**: You can write fast without leaving the keyboard. There are no menus to click.

### The cons of Markdown

Markdown has a few limits.

- **Lack of standards**: Markdown has no single set of rules. Many versions exist, like GFM and CommonMark. A feature
  that works in one app might fail in another.
- **Limited features**: Markdown lacks tools for complex layouts. Tables and diagrams are not in the main rules.
  Different apps show them in different ways.
- **No built-in styling**: Markdown separates content from style. This separation means you cannot change fonts, colors,
  or layouts in the file.

### Common mistakes to avoid

Most errors come from small mistakes. Here are a few:

- **Forgetting blank lines**: Markdown uses empty lines to separate text blocks. Leaving them out is a common mistake
  that breaks formatting.
- **Broken links**: You can easily mix up brackets. Always check your links.
- **Unformatted code**: Add a language name after the backticks (like js). Without it, the code renders as plain text.
- **Bad spacing**: Nested lists need correct spaces. Wrong spacing breaks the list.

### Tips for better Markdown

- **Use a linter**: Linters check files for errors and enforce a clean style. These tools are helpful when writing with
  a team.
- **Learn your flavor**: Learn the rules of the site you use, like GitHub. Doing so helps you use all features and avoid
  errors.
- **Keep tables simple**: Tables are hard to write in plain text. Use a simple list instead.
- **Use HTML**: Markdown supports HTML. You can use raw HTML code for complex tasks.

Markdown is simple and useful. Knowing its features helps you write well.
